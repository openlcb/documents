The purpose of the OpenLCB™ project is to develop a new Layout Control Bus for use on Model Railroads.

Development Area

This section describes the organization of the OpenLCB development areas. See also the SourceForge.net http://sourceforge.net/projects/openlcb/.

File Tree Organization

This area contains a number of subdirectories:

specs
Draft specifications and technical notes that are in the process or have been formalized. This is the place to start when looking for protocol technical information.
documents
Working and development documents; generally, these will move to the specs directory as they become more formalized.
prototypes
Prototype code, organized by language and sometimes by project.
srpdrafts
At one point, we were developing Standards and Recommended Practices documents for discussion and eventual submission to the NMRA in this directory. For the current status and why we no longer are, see here.
web
Bits for the web site: logos, etc.

Only commonly-held formats are to be used for files. E.g. documentation should be in HTML 4.0, preferably edited with a tool like Open Office that preserves minimal formatting. Drafts should be in DocBook so that we can easily convert them to multiple formats; there are several free DocBook editing tools available.

Access

The content of this development area, including code, documents and web pages, is kept in the OpenLCB SourceForge.net SVN repository in the “trunk” directory. To check it out, please do:

svn co https://openlcb.svn.sourceforge.net/svnroot/openlcb openlcb

For more information on the SVN source code management tool, please see the SVN website and the online SVN book.
